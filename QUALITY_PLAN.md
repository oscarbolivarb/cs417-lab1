# Software Quality Plan

## 1. Usability 

**Definition:** means that the software is easy for users to learn and to be able to use the software application. The users should be able to complete tasks without any problems.

**Why it matters:** this is important because people use the software more when it is easy to use but if it is difficult to use, then users get frustrated and do not use it.

**Planning impact:** During the planning stage, the development team should think about the users and create a design with a simple interface. The stakeholders will also expect that the software is easy to use especially for different types of users.

## 2. Security

**Definition:** means that  the software is protected and users information is safe by unauthorized attacks or access.

**Why it matters:** security matters because users trust the software companies with their personal information to keep it safe. A strong security also helps reduce any cyber threats or data breaches.

**Planning impact:** the team should plan the security from the beginning of the project and this includes creating security logging, protecting data and testing the system for any security problems.

## 3. Maintainability

**Definition:** means that the software is easy to update, fix and improve after it has been released. 

**Why it matters:** software often needs to be updated, by fixing any bugs, improve any performance issues or add new features. If the software is easy to maintain, then these changes can be made faster with fewer problems.

**Planning impact:** during the project's planning, the developers should be able create to create a clear design and write organized code. This will make it easier to update the software in the future and meet stakeholders expectations.
